# 日報格式 — 撰寫指南

## 平台：微信公眾號

### 整體結構

```
[開場語] — 1-2 句，設定今天的基調
[頭條] — 今天最重要的 1 條，多寫幾句
[正文] — 5-7 條精選新聞
[結尾語] — 1 句話收尾 + 互動引導
```

### 開場語

每天不同，但風格一致。可以的模式：
- 日期 + 一句話概括今天的主旋律
- 用今天最大的新聞作為開頭的 hook
- 季節/節日/行業事件的關聯

範例：
> 3 月 20 日，週四。今天 AI 圈又炸了，不過這次不是技術突破，是錢的問題。

### 每條新聞的寫法

```
### [簡潔標題，< 20 字]

[核心事實，1-2 句] [點評或 so what，1 句]

> 延伸：[可選，如果有相關背景值得補充，1 句]
```

**寫法原則**：
- 事實先行：誰、做了什麼、數字是多少
- 點評要有態度：不是「值得關注」這種廢話，而是「這意味著 XX」
- 每條 50-150 字，不超過 200 字
- 標題用動詞開頭更有力（「砍掉」比「關於裁撤」好）

**好的日報條目範例**：
> ### OpenAI 推出企業版 API 定價，降幅達 40%
> OpenAI 宣布 GPT-4 API 價格下調 40%，同時推出企業級 SLA 和專屬支持。這不是在讓利，是在搶 AWS Bedrock 和 Google Cloud 的企業客戶。
> > 延伸：上週 Anthropic 剛推出類似的企業方案，價格戰正式開打。

**不好的日報條目**（避免）：
> ### 一則關於 AI 定價的消息
> 近日，OpenAI 方面表示將對其 API 進行價格調整。具體來說，GPT-4 系列的 API 調用費用將有所下降。業內人士認為這一舉措可能會對市場產生一定影響。

### 排序邏輯

1. 頭條（最重磅，或最多人關心的）
2. 同類話題就近排列（如兩條都是融資新聞，放一起）
3. 重的在前，輕的在後
4. 最後一條可以放稍微輕鬆的（彩蛋感）

### 結尾語

簡短，帶互動：
- 「今天哪條最讓你意外？留言聊聊。」
- 「以上就是今天的精選。明天見。」
- 提問式結尾比陳述式更好

### 發布建議

- **時間**：早上 7:30-8:30（通勤時間）或 中午 12:00-13:00（午休）
- **標題格式**：「[日期] 商業日報 | [今日最大看點]」
- **封面圖**：統一風格模板，每天換配色或數字
