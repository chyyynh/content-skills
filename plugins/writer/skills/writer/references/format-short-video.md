# 短影片腳本格式 — 撰寫指南

## 平台：小紅書 / 抖音

### 腳本結構（黃金 60-180 秒）

```
[Hook]       — 0-3 秒，一句話抓住注意力
[痛點/好奇]   — 3-10 秒，建立「為什麼你要聽」
[核心內容]    — 10-120 秒，正文（2-5 個要點）
[結論/金句]   — 最後 10-15 秒，一句話總結
[CTA]        — 最後 5 秒，引導互動
```

### Hook 公式

Hook 決定 80% 的完播率。以下是經過驗證有效的 Hook 模式：

**數字衝擊型**：
- 「一家公司靠 XX 功能，3 個月拿了 100 萬用戶」
- 「他 25 歲，公司估值 10 億，但只有 3 個員工」

**反直覺型**：
- 「為什麼越努力融資的公司越容易倒？」
- 「投資人最怕聽到這句話，但 90% 的創業者都在說」

**身份代入型**：
- 「如果你在考慮辭職創業，先看完這個」
- 「年薪百萬的產品經理都在偷偷學的一件事」

**時效爆點型**：
- 「昨天 XX 發生了一件大事，所有創業者都該知道」
- 「剛剛，XX 公司宣布了一個改變行業的決定」

### 核心內容的組織方式

**列表式**（最安全，適合新手）：
```
「今天講 3 個 XX 的關鍵點」
第一個...
第二個...
第三個...
```

**對比式**（有衝擊力）：
```
「A 公司做了 XX，成功了；B 公司也做了 XX，卻失敗了。差別在哪？」
→ 展開對比
→ 揭示關鍵差異
```

**故事線式**（最吸引人但最難）：
```
背景設定 → 轉折點 → 高潮 → 啟示
```

### 口播腳本 vs 混剪腳本

**口播**（出鏡講述）：
- 語言更口語，允許語氣詞
- 可以用手勢提示（「第一點☝️」）
- 在腳本中標注 [畫面提示] 和 [字幕強調]

```
[面對鏡頭] 你知道為什麼 90% 的 SaaS 創業公司活不過兩年嗎？
[字幕：90% 活不過兩年] 不是因為產品不好，是因為——
[停頓 1 秒，表情變認真] 他們搞錯了一件事。
[字幕：定價！] 就是定價。
```

**混剪**（畫面+旁白）：
- 需要標注每段旁白對應什麼畫面
- 旁白更精煉，靠畫面補充信息

```
[畫面：XX 公司辦公室空景] 旁白：上個月，這家估值 50 億的公司，辦公室突然空了一半。
[畫面：新聞截圖] 旁白：裁員 30%，沒有任何預警。
[畫面：CEO 照片] 旁白：CEO 說是「戰略調整」，但真正的原因是...
```

### 平台差異

| | 小紅書 | 抖音 |
|---|---|---|
| 最佳時長 | 60-90 秒 | 30-60 秒（新號），60-180 秒（老號） |
| 風格偏好 | 精緻、有設計感、知識密度高 | 快節奏、情緒化、強 hook |
| 字幕 | 必須有，排版要好看 | 必須有，大字醒目 |
| 封面 | 文字封面，關鍵數據/問題 | 人物表情 + 大字標題 |
| CTA | 「收藏起來慢慢看」「關注我學更多」 | 「點贊這條」「評論區聊聊」 |

### 輸出格式

最終腳本包含：

```markdown
## 影片腳本：[標題]

**平台**：小紅書 / 抖音
**預估時長**：XX 秒
**類型**：口播 / 混剪

### 封面文案
[封面上的文字]

### 標題（發布用）
[平台發布時的標題/第一行文案]

### 腳本正文
[按時間軸排列的完整腳本，含畫面提示和字幕標注]

### Hashtag 建議
#標籤1 #標籤2 #標籤3

### 發布建議
- 最佳時段：XX
- 互動引導：XX
```

### 常見陷阱

- ❌ 開頭說「大家好，今天我們來聊...」→ 3 秒內就被滑走
- ❌ 堆砌太多信息，觀眾消化不了 → 一條影片只講 1 個核心觀點
- ❌ 結尾突然結束，沒有收束感 → 最後一句要有力
- ❌ 用書面語口播 → 寫完後用「念出來」測試，念起來順才行
