# 短影片腳本公式庫

clip 單獨使用（沒有 writer 腳本）時的內建參考。根據剪輯內容選擇合適的公式。

## 腳本結構

```
[Hook 封面+配音]  0-3 秒   — 一句話鉤住觀眾，配合封面圖
[痛點/好奇心]     3-10 秒  — 為什麼你該繼續看（可融入主片段開頭）
[核心內容]        10-50 秒 — 重點剪輯片段，帶翻譯字幕
[CTA]            最後 3-5 秒 — 引導互動（可選）
```

短影片總時長控制在 **30-60 秒**，完播率最高。

---

## Hook 公式（前 3 秒，決定 80% 完播率）

### 1. 懸念型 — "你知道 _____ 嗎？"

製造知識缺口，觀眾必須看完才能填補。

| 模板 | 範例 |
|------|------|
| 你知道 [意外事實] 嗎？ | "你知道 OpenAI 昨天偷偷改了什麼嗎？" |
| [主體] 有一個秘密，99% 的人不知道 | "GPT-5 有個隱藏功能，99% 的人不知道" |
| 你猜 [事件] 背後真正的原因是什麼？ | "你猜 Google 裁員背後真正的原因是什麼？" |

**適用場景**：內幕消息、技術揭秘、反常識事實
**完播率表現**：70%+ completion rates

### 2. 顛覆型 — "你以為的 _____ 全錯了"

挑戰既有認知，觸發「我要看看為什麼」的心理。

| 模板 | 範例 |
|------|------|
| 你以為 [常見認知]？完全搞反了 | "你以為 AI 會取代程式設計師？完全搞反了" |
| 關於 [話題]，所有人都搞錯了一件事 | "關於 Prompt Engineering，所有人都搞錯了一件事" |
| [權威來源] 說的這句話，顛覆了整個行業 | "Sam Altman 說的這句話，顛覆了整個 AI 行業" |

**適用場景**：爭議觀點、行業變革、反直覺發現
**互動表現**：高評論率，觀眾想辯論

### 3. 緊急型 — "在你 _____ 之前，先看這個"

製造「不看會虧」的急迫感。

| 模板 | 範例 |
|------|------|
| 在你 [常見行為] 之前，先看這個 | "在你開始用 Cursor 之前，先看這個" |
| [時間] 之前你必須知道這件事 | "下週之前你必須知道這件事" |
| 如果你正在 [行為]，立刻停下來 | "如果你正在用 ChatGPT 寫程式碼，立刻停下來" |

**適用場景**：工具評測、避坑指南、時效性強的消息

### 4. 數據型 — "只有 X% 的人知道 _____"

用具體數字製造衝擊力和排他感。

| 模板 | 範例 |
|------|------|
| 只有 [小數字]% 的人知道 [有價值的事] | "只有 1% 的人知道這個免費 AI 工具" |
| [大數字] 的人都在犯這個錯 | "90% 用 AI 的人都在犯這個錯" |
| [主體] 靠 [方法]，[時間] 內達到 [成果] | "這家公司靠一個 AI Agent，3 個月省了 500 萬" |

**適用場景**：數據驅動的新聞、產品對比、趨勢分析
**分享表現**：被收藏/分享的機率是無結構內容的 2.5 倍

### 5. 錯誤型 — "X 個 _____ 的常見錯誤"

人天然會想知道自己是不是犯了這些錯。

| 模板 | 範例 |
|------|------|
| [動作] 的人都在犯的 [數字] 個錯 | "用 ChatGPT 的人都在犯這 3 個錯" |
| 別再 [錯誤做法] 了，試試這個 | "別再手動寫 Prompt 了，試試這個" |
| 為什麼你的 [目標] 一直沒效果？因為 [原因] | "為什麼你的 AI 產品一直沒人用？因為你忽略了這一點" |

**適用場景**：教學內容、工具使用技巧、行業常見誤區

### 6. 轉變型 — "我從 X 到 Y 只用了 _____"

展示具體轉變，觀眾想知道怎麼做到的。

| 模板 | 範例 |
|------|------|
| 從 [差狀態] 到 [好狀態]，只用了 [方法/時間] | "從月薪 3 萬到年入百萬，他只做了一件事" |
| [方法] 讓我的 [指標] 翻了 [倍數] | "這個方法讓我的工作效率翻了 3 倍" |
| [時間] 前我還在 [舊狀態]，現在 [新狀態] | "半年前我還在手動剪片，現在 AI 幫我全搞定" |

**適用場景**：個人經驗、工具推薦、方法論分享

### 7. FOMO 型 — "_____ 你聽說了嗎？"

利用怕錯過的心理，暗示「別人都知道了，你還不知道？」

| 模板 | 範例 |
|------|------|
| [新事物] 你聽說了嗎？ | "Google 剛發布的這個東西你聽說了嗎？" |
| 全網都在討論 [話題]，到底怎麼回事？ | "全網都在討論 Claude Code，到底怎麼回事？" |
| [行業] 的人都在偷偷用 [工具/方法] | "矽谷的工程師都在偷偷用這個 AI 工具" |

**適用場景**：新產品發布、熱門事件、趨勢報導
**評論表現**：評論率高 83%

### 8. 直球型 — "X 分鐘告訴你 _____"

不繞彎，直接告訴觀眾他能得到什麼。

| 模板 | 範例 |
|------|------|
| [時間] 搞懂 [複雜概念] | "60 秒搞懂 AI Agent 到底是什麼" |
| 一條影片講清楚 [話題] | "一條影片講清楚 MCP 協議為什麼重要" |
| [話題] 最完整的懶人包 | "GPT-5 最完整的懶人包，看這一條就夠" |

**適用場景**：科普解釋、新聞懶人包、技術介紹

## CTA 公式

### 小紅書風格
- "覺得有用的話，收藏起來慢慢看 👆"
- "關注我，每天帶你看 AI 最前線"
- "你怎麼看？評論區聊聊"

### 抖音風格
- "點贊這條，下次更新不會錯過"
- "評論區告訴我你最想了解什麼"
- "關注我，明天繼續聊"

### YouTube Shorts 風格
- "Subscribe for more AI insights"
- "想看更多？訂閱不迷路"
- "你覺得呢？留言告訴我"

## 封面設計原則

### 文字封面（小紅書偏好）
- 一行主標題（10 字以內），大字醒目
- 可選副標題（人名、來源）
- 背景暗化或純色，文字高對比
- 範例：主標 "GPT-5 來了" + 副標 "Sam Altman 親自宣布"

### 人物封面（抖音偏好）
- 人物表情/動作為主體
- 大字標題疊在畫面上
- 情緒張力要強（驚訝、認真、興奮）

### AI 生成封面
用 image plugin 生成時，prompt 要包含：
- 主題相關的視覺元素
- 9:16 構圖，文字區域留白
- 風格：cinematic / editorial / dramatic
- 明確指定文字內容和位置

## 平台參數速查

| 平台 | Hook 時間 | 最佳時長 | 風格 | 字幕 |
|------|----------|---------|------|------|
| 抖音 | ≤ 2 秒 | 30-60 秒 | 高能量、快節奏、情緒化 | 大字醒目 |
| 小紅書 | ≤ 3 秒 | 60-90 秒 | 精緻、知識密度高、有設計感 | 排版精美 |
| YouTube Shorts | 3-5 秒 | 30-60 秒 | 結構化敘事、資訊導向 | 清晰可讀 |

## 留住觀眾的剪輯技巧

- **每 3-5 秒切一次畫面**：換鏡頭、加文字彈出、插入 B-roll
- **85% 的人靜音觀看**：字幕不是可選的，是必須的
- **用小學六年級的語言**：簡單直接，觀看量翻 2 倍
- **開頭不要打招呼**：「大家好，今天我們來聊...」= 立刻被滑走
- **結尾不要突然斷掉**：最後一句要有力，給觀眾一個「記住點」
