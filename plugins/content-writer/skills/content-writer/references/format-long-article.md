# 長文草稿格式 — 撰寫指南

## 平台：微信公眾號 / Twitter Thread

---

## 微信公眾號長文

### 目標
2000-5000 字的深度分析，讓讀者讀完後有「這篇值得收藏」的感覺。

### 結構模板

```
[標題] — 引發好奇，不劇透結論
[導語] — 100-200 字，設定問題框架
[正文]
  ├── 第一部分：現象描述（發生了什麼）
  ├── 第二部分：深層分析（為什麼會這樣）
  ├── 第三部分：延伸思考（這意味著什麼）
  └── 第四部分：行動建議（所以呢）
[結尾] — 回扣主題，留下思考空間
```

### 標題的寫法

**好標題的特徵**：
- 有具體信息（數字、公司名、事件）
- 暗示有獨到觀點（不是人人都知道的事）
- 控制在 20-30 字

**標題公式**：
| 公式 | 範例 |
|------|------|
| [事件] + [反常識解讀] | 「Notion 估值腰斬：筆記工具的終局不是 AI」 |
| [數字] + [趨勢判斷] | 「3 家獨角獸同時裁員，SaaS 的冬天才剛開始」 |
| [人物] + [出乎意料的決策] | 「他放棄了 10 億估值的公司，去做了一個免費產品」 |
| [問題] | 「為什麼中國的 SaaS 公司總是做不大？」 |

### 導語的寫法

導語要在 3 秒內讓讀者決定繼續看下去。有效的模式：

**場景代入**：
> 如果你是一個 SaaS 創業者，過去三個月你可能已經注意到一個讓人不安的趨勢：你的客戶開始問你，「你們有 AI 功能嗎？」

**反常識開場**：
> 所有人都在說 AI 會顛覆一切，但最先被 AI 顛覆的可能不是你想的那些行業，而是 AI 行業自己。

**數據衝擊**：
> 過去 6 個月，全球有 47 家 AI 創業公司倒閉，其中 32 家在倒閉前 12 個月內曾獲得融資。

### 正文寫作原則

**每個段落的結構**：
- 開頭：這一段要講什麼（一句話）
- 中間：展開論述（數據、案例、邏輯推演）
- 結尾：這意味著什麼（過渡到下一段）

**保持節奏感**：
- 長段落（200-300 字分析）和短段落（1-2 句觀點）交替
- 每 500 字左右放一個「呼吸點」（小標題、引用、數據圖表的位置）
- 不要連續三段以上都是同一種節奏

**引用和數據**：
- 每個重要觀點至少有一個數據或案例支撐
- 數據來源要交代（「根據 CB Insights 的數據...」）
- 用具體案例 > 抽象數據 > 純觀點

### 結尾的寫法

**不要**：
- 「總之，XXX 值得我們關注」（廢話）
- 突然結束，沒有收束
- 過度樂觀或悲觀的預測

**要**：
- 回到導語設定的問題，給出你的回答
- 留一個開放性的思考給讀者
- 如果合適，提供一個可操作的建議

## Twitter Thread

### 目標
10-25 則推文組成的長 thread，節奏快、觀點密、易傳播。

### 結構

```
Tweet 1  — Hook（最重要，決定是否展開看）
Tweet 2  — 背景/問題設定
Tweet 3-N — 核心論點（每則一個要點）
倒數第二 — 總結/最重要的 takeaway
最後一則 — CTA + 自我介紹（可選）
```

### 每則推文的寫法

**Tweet 1（Hook）**：
- 必須獨立成立（不依賴後續推文）
- 280 字以內（中文約 140 字）
- 讓人想點「展開全文」

範例：
> 我研究了 50 家 AI 創業公司的定價策略。
> 發現一個反直覺的規律：
> 定價最貴的公司，存活率反而最高。
> 為什麼？🧵

**正文推文**：
- 每則只講一個要點
- 第一句就是結論，後面是支撐
- 適當使用 emoji 做視覺分隔（不要過度）
- 不同推文之間的邏輯連接要明確

**最後一則**：
- 總結 thread 的核心觀點（1-2 句）
- 引導互動（「你覺得呢？」「你的經驗是什麼？」）
- 可以引導回推第一則（增加曝光）

### 中英文差異

| | 中文 Thread | 英文 Thread |
|---|---|---|
| 每則長度 | 100-140 字 | 200-280 字 |
| 總推文數 | 10-20 則 | 8-15 則 |
| 風格 | 可以更口語化 | 保持專業但不學術 |
| Emoji | 適度，做分隔用 | 較常用，做標記用 |

### 輸出格式

```markdown
## Twitter Thread：[主題]

**語言**：中文 / 英文
**預估推文數**：XX 則

### Thread

**1/**
[Hook 推文]

**2/**
[背景]

**3/**
[論點 1]

...

**N/**
[CTA]

### 發布建議
- 最佳時段：XX
- 是否需要配圖：XX
- 建議互動方式：XX
```

## 通用原則（所有長文）

1. **先寫大綱再展開**：讓用戶確認結構後再動筆，避免大幅返工
2. **每段都要有信息增量**：如果刪掉這一段文章依然完整，那這段可能不需要
3. **寫完後的自檢**：
   - 刪掉所有「其實」「事實上」「眾所周知」等填充詞
   - 檢查每個觀點是否有支撐
   - 用「一句話能不能概括這篇文章」測試主題是否清晰
